What is Git?

Git is a Version Control System (VCS).

Think of it as a save game feature in a video game.

Without Git:

Project
│
├── code.py
├── app.py

If you accidentally delete your code, it's gone.

With Git:

Version 1
      │
      ▼
Version 2
      │
      ▼
Version 3
      │
      ▼
Latest

You can go back to any previous version.

Git stores your project on your computer.

GitHub stores your Git repository on the cloud.

Key takeaway

Don't think of Git as a list of commands. Think of it as a workflow:

Create or modify files (Working Directory)
Stage the changes (git add)
Save a snapshot (git commit)
Upload it to GitHub (git push)

Once this workflow becomes second nature, learning advanced topics like branches, pull requests, merge conflicts, rebasing, and collaborative development will be much easier. Since your goal is AI Engineering followed by Full-Stack Engineering, mastering this workflow is an essential foundation because virtually every professional AI project uses Git.

These are some of the most important Git commands you'll use every day as an AI Engineer or Full-Stack Engineer. I'll explain each one with a real-world analogy and practical examples.

1. git merge
What is it?

git merge combines changes from one branch into another.

Imagine two developers are working on the same project.

                main
                  │
                  │
          -----------------
          │               │
     login branch    payment branch

Developer A creates the login feature.

Developer B creates the payment feature.

Now you want both features in main.

git checkout main
git merge login

Git copies all the commits from the login branch into main.

Result:

main

Login Feature
Payment Feature
Example

git checkout -b login

Edit

login.py

git commit -m "Added Login"

Go back

Merge

Now everything from the login branch exists in main.

Why do companies use merge?

Because every developer works separately.
