Example 1: for Loop – Processing Multiple Documents (RAG Project)

Imagine you're building a chatbot that answers questions from PDF files.

Your project folder:

documents/
│── ai.pdf
│── python.pdf
│── langgraph.pdf
│── fastapi.pdf

You need to read every PDF.

documents = [
    "ai.pdf",
    "python.pdf",
    "langgraph.pdf",
    "fastapi.pdf"
]

for i in documents:
    print(f"Reading {i}")

Output

Reading ai.pdf
Reading python.pdf
Reading langgraph.pdf
Reading fastapi.pdf
What's happening?

The loop processes one document at a time.

Iteration 1 → ai.pdf
Iteration 2 → python.pdf
Iteration 3 → langgraph.pdf
Iteration 4 → fastapi.pdf

This is exactly how RAG systems process multiple documents.

Example 3: AI Training (Epochs)

Machine learning models don't learn in one pass.

They repeat the training process many times.

epochs = 10

for epoch in range(epochs):

    print(f"Training Epoch {epoch+1}")

Output

Training Epoch 1
Training Epoch 2
...
Training Epoch 10

Real training code looks similar.

for epoch in range(10):

    train_model()

    calculate_loss()

    save_weights()

This is how PyTorch and TensorFlow training loops work.

range() is simply a way to generate a sequence of numbers, often used to repeat an action a fixed number of times.

    print("Training Model")

Where you'll use them in your AI Engineer journey
Project	for	while	range()
RAG	Read every document	Wait for user query	Chunk documents
LangGraph	Execute workflow steps	Wait for approval	Retry limited times
FastAPI	Process uploaded files	Wait for background task	Loop through results
PyTorch	Process every training batch	Early stopping conditions	Train for N epochs
AI Agents	Run tools one by one	Continue conversation	Retry tool calls
LLM Apps	Process prompts	Chat until user exits	Generate multiple outputs
A simple rule to remember
for → "I know what I need to process (a list of items)."
range() → "I know how many times I want to repeat something."
while → "I don't know when I'll be finished; I'll stop only when a condition changes."

If you keep these three questions in mind, choosing between for, range(), and while becomes much more intuitive in real AI projects.
